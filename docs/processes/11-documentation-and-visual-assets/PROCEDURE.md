# Documentation And Visual Assets Procedure

## Purpose

Keep Smartcito process documentation clear, current, and easy for new
developers to follow.

## Scope

This procedure applies to process documents, runbooks, workflows, screenshots,
diagrams, and PNG visual assets stored under [../](../).

## Procedure

1. Choose the correct process subfolder before writing a new procedure.
2. Use a clear title, purpose, scope, owners, prerequisites, procedure steps,
   validation checklist, evidence section, and related links.
3. Write steps as direct actions that a new developer can follow without hidden
   team context.
4. Save all required visuals as PNG files under [../assets/png](../assets/png).
5. Use professional technical visuals with readable labels and restrained colors.
6. Add every PNG file to [../assets/PNG_ASSET_MANIFEST.md](../assets/PNG_ASSET_MANIFEST.md).
7. Link the visual from the procedure section it supports.
8. Review the document for broken links, stale commands, ambiguous ownership,
   and missing validation steps.

## PNG Visual Requirements

- Use PNG format for all process images.
- Prefer diagrams, screenshots, and flow visuals that clarify the actual task.
- Avoid decorative images that do not teach or verify a process.
- Keep image text concise and legible.
- Use lowercase hyphenated filenames.

## Validation Checklist

- Procedure is in the correct subfolder.
- Steps are complete enough for a new developer.
- PNG assets open successfully.
- Manifest is updated.
- Links resolve from the rendered Markdown document.
